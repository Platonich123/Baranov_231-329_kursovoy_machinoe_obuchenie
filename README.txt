# Интерпретируемость моделей для крипто-аудитов

Курсовой проект по дисциплине **«Методы машинного обучения»**.  
Вариант 5: **Интерпретируемость моделей для крипто-аудитов**.

## Автор

Баранов Иван, группа 231-329.

## Цель проекта

Построить модель машинного обучения для выявления подозрительных Ethereum-объектов и объяснить решения модели с помощью методов интерпретируемости.

## Датасет

Используется открытый датасет Kaggle:

**Ethereum Fraud Detection Dataset**  
https://www.kaggle.com/datasets/vagifa/ethereum-frauddetection-dataset

В датасете каждая строка соответствует Ethereum-адресу. Признаки описывают транзакционную активность адреса: количество отправленных и полученных транзакций, объемы ETH, операции с ERC20-токенами, уникальные адреса и временные характеристики.

Целевой признак:

- `FLAG = 0` — нормальный Ethereum-объект;
- `FLAG = 1` — мошеннический или подозрительный Ethereum-объект.

## Использованные модели

В работе были обучены и сравнены 5 моделей классификации:

1. Logistic Regression;
2. Decision Tree;
3. Random Forest;
4. Gradient Boosting;
5. KNN.

Дополнительно для анализа без учителя использован **Isolation Forest**.

## Метрики качества

Для оценки моделей использовались:

- Accuracy;
- Precision;
- Recall;
- F1-score;
- ROC-AUC.

Наиболее важными для задачи выявления мошенничества являются `Recall` и `F1-score`, так как необходимо находить как можно больше подозрительных объектов и учитывать баланс между полнотой и точностью.

## Лучшая модель

Лучший результат показала модель **Gradient Boosting**. После подбора гиперпараметров с помощью GridSearchCV были получены следующие значения:

- Accuracy ≈ 0.9837;
- Precision ≈ 0.9879;
- Recall ≈ 0.9381;
- F1-score ≈ 0.9624;
- ROC-AUC ≈ 0.9974.

## Интерпретируемость

Для объяснения решений модели использовались:

- Feature Importance;
- SHAP summary plot;
- SHAP bar plot;
- SHAP waterfall plot для отдельного объекта.

Это позволяет не только получить прогноз `fraud / non-fraud`, но и понять, какие признаки транзакционной активности повлияли на решение модели.
